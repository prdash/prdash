## Objective

The PR description in the detail pane is currently rendered as plain text. Use Textual's built-in `Markdown` widget to render it as formatted Markdown, giving the user a much more readable view of PR bodies that contain headings, lists, code blocks, links, etc.

## Background

Textual provides `textual.widgets.Markdown` which renders GitHub-flavored Markdown in the terminal. The detail pane's description section (`#detail-description`) currently uses a `Static` widget with the raw `pr.body` string. Swapping this to a `Markdown` widget is a relatively contained change.

## Requirements

1. **Replace the description `Static` with a `Markdown` widget**: The `#detail-description` section in `DetailPaneWidget` should use `textual.widgets.Markdown` instead of `Static`.
2. **Update content via `Markdown.update()`**: When `show_pr()` is called, update the Markdown widget with `pr.body`.
3. **Handle empty/missing body**: If `pr.body` is empty or `None`, display a short fallback like `*No description provided.*` (rendered as italic).
4. **Preserve other sections as-is**: Only the description section changes; metadata, labels, reviewers, checks, and timeline remain `Static` widgets.
5. **CSS adjustments**: Ensure the Markdown widget fits within the detail scroll container and doesn't break the layout. Textual's Markdown widget may need minor CSS tweaks for padding/margins.
