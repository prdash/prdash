## Objective

Allow users to remap keybindings via TOML config, using Textual's binding system.

## Background

Users have different keyboard preferences and may want to avoid conflicts with their terminal emulator's shortcuts. A `[keybindings]` config section lets users override default bindings while keeping sensible defaults.

## Requirements

1. **Config schema**: Add `[keybindings]` table in TOML config mapping action names to key strings (e.g., `refresh = "ctrl+r"`)
2. **Binding override**: On app startup, apply user-defined bindings over the defaults using Textual's binding API
3. **Validation**: Validate key strings on config load; warn on invalid keys or conflicting bindings
4. **Help screen update**: Ensure the help/keybinding reference screen reflects user-customized bindings
5. **Tests**: Test binding override logic, validation, and conflict detection
