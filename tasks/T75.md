## Objective

Add an optional `nerd_font = true` config setting that swaps the default Unicode symbols for Nerd Font glyphs, giving users with Nerd Fonts installed a more polished icon experience.

## Background

T74 introduces Unicode symbol constants for CI, Review, and other status indicators. Users who have a Nerd Font installed in their terminal (common among developers using tools like starship, powerlevel10k, lazygit) can get richer glyphs. This task adds a config toggle that selects the icon set at startup.

## Requirements

1. **Config field:** Add `nerd_font: bool = False` to `AppConfig`
2. **Icon sets:** Define a Nerd Font icon dict alongside the Unicode icon dict from T74, mapping the same keys to Nerd Font glyphs:
   - CI passing: `` (U+F058)
   - CI failing: `󰅙` (U+F0159)
   - CI pending: `` (U+E641)
   - Review approved: `󰄬` (U+F012C)
   - Review changes: `` (U+EB43)
   - Comment: `` (U+F27B)
   - PR open: `` (U+F407)
   - PR draft: `` (U+EBDB)
3. **Icon selection:** At app startup, select the icon dict based on `config.nerd_font` and pass it through to widgets
4. **TOML serialization:** Include `nerd_font` in config output when `true`
5. **Tests:** Verify both icon sets are complete and that the toggle selects the correct set

## Files

## Acceptance Criteria

- [ ] `nerd_font = false` (default) uses Unicode symbols from T74
- [ ] `nerd_font = true` uses Nerd Font glyphs
- [ ] Config field round-trips through load/save correctly
- [ ] Both icon dicts have identical keys
- [ ] PR rows render correctly with either icon set
