# T47 - Persistent Group Collapse State (save/restore across sessions)

## Objective

Remember which PR groups are collapsed across sessions using a lightweight JSON state file, so users don't have to re-collapse groups every time the app starts.

## Background

The PR list has collapsible groups (e.g., "My PRs", "Review Requested", "Assigned"). When users collapse groups they don't care about, that preference is lost on app restart. Persisting this state provides a smoother experience. A separate state file (not the config TOML) is appropriate since this is ephemeral UI state rather than configuration.

## Requirements

1. **State module**: Create `state.py` with `load_state()` and `save_state()` functions
2. **State file location**: `~/.config/prdash/state.json` (same directory as config TOML)
3. **State schema**: JSON object with `collapsed_groups: list[str]` (group labels that are collapsed)
4. **Atomic writes**: Write to a temporary file then rename to avoid corruption on crash
5. **Load on construction**: `PRListWidget` reads collapsed state on initialization and applies it when building the list
6. **Save on toggle**: When a group is collapsed/expanded, save the updated state
7. **Graceful fallback**: If the state file is missing, corrupted, or unreadable, default to all groups expanded
8. **Tests**: Test load/save round-trip, missing file fallback, corrupted file fallback, atomic write behavior
