## Objective

Restructure PR list rows to use a two-line layout with fixed-width right-aligned status columns, replacing the current inline text-label approach with a clean, uniform grid inspired by gh-dash.

## Background

The current PR row layout concatenates metadata inline with `·` separators (`@alice · 2d · CI:pass · Rev:ok · +350/-120 · 5💬`). This produces ragged, hard-to-scan rows because text labels vary in length. gh-dash solves this with fixed-width columns for status indicators, right-aligned at the row edge, creating perfect vertical alignment across all rows.

## Requirements

### Row Layout

1. **Two-line layout per PR row:**
   - **Line 1 (metadata):** `repo/name #123 by @author · 2d ago` in dim/secondary text, left-aligned
   - **Line 2 (title):** PR title in bold primary text, left-aligned
2. **Right-aligned status columns** on Line 1, fixed-width:
   - CI status icon (1 char, colored)
   - Review status icon (1 char, colored)
   - Size indicator `+N/-N` (fixed width, ~12 chars, green/red)
   - Comment count icon + number (fixed width, ~4 chars)
3. **Badges on Line 2** (after title, inline): DRAFT (cyan), CONFLICT/BLOCKED/BEHIND as before
4. **Ellipsis truncation:** Titles and metadata that exceed available width are truncated with `…` rather than wrapping, keeping each row exactly 2 lines tall
5. **New-PR marker** (bold dot) remains as left-most element
6. **BranchRow** follows same layout pattern for consistency

### Unicode Symbol Set (Default)

7. **CI status icons:**
   - Passing: `✓` (green)
   - Failing: `✗` (red)
   - Pending: `◍` (yellow)
   - None: `–` (dim)
8. **Review status icons:**
   - Approved: `✓` (green)
   - Changes requested: `✗` (red)
   - Pending: `○` (yellow)
   - None: `–` (dim)
9. **Comment icon:** `💬` or `✉` with count
10. **Icon constants** defined in a dedicated dict/module so T75 can swap them

### Widget Structure

11. **PRRow.compose():** Use a `Horizontal` container with:
    - Left section: `Static` with metadata line + title line (takes `1fr` width)
    - Right section: `Static` with fixed-width status columns (right-aligned, fixed character width)
12. **CSS:** Fixed widths for the right-column container, right-aligned text
13. **Approved-by-me** and **ready-to-merge** background highlights continue to work

### Testing

14. Existing PR list tests updated to match new layout structure
15. New tests verifying:
    - Status icons render correctly for each state
    - Right-aligned columns maintain fixed width
    - Ellipsis truncation when title exceeds available space
    - Badge rendering (DRAFT, CONFLICT) on line 2
