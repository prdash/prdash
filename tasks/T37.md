## Objective

Fix the `ci_status` computed property so that PRs with `NEUTRAL` or `SKIPPED` check conclusions show `CI:pass` instead of `CI:pend`, and PRs with `CANCELLED`, `TIMED_OUT`, `ACTION_REQUIRED`, `STALE`, or `STARTUP_FAILURE` conclusions show `CI:fail`.

## Background

The `ci_status` logic only treated `SUCCESS` as passing and `FAILURE` as failing. All other completed conclusions (like `NEUTRAL`, `SKIPPED`) fell through to `"pending"`, which is incorrect — these are valid passing states. Similarly, conclusions like `CANCELLED` and `TIMED_OUT` were not recognized as failures.

## Requirements

1. **Pass conclusions**: Add module-level `_CI_PASS_CONCLUSIONS` frozenset containing `SUCCESS`, `NEUTRAL`, `SKIPPED`
2. **Fail conclusions**: Add module-level `_CI_FAIL_CONCLUSIONS` frozenset containing `FAILURE`, `CANCELLED`, `TIMED_OUT`, `ACTION_REQUIRED`, `STALE`, `STARTUP_FAILURE`
3. **Updated logic**: Use set membership (`in`) instead of string equality in `ci_status` property
4. **Pending**: `None` conclusions (in-progress checks) don't match `_CI_PASS_CONCLUSIONS`, so still return "pending"
5. **Tests**: Cover all new conclusion types as passing or failing
