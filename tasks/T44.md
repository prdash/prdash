## Objective

Detect and surface meaningful changes on refresh: new PRs appearing, CI status transitions, and review status changes, using Textual's built-in toast notification system.

## Background

When the dashboard auto-refreshes, users have no way to know what changed without scanning the entire list. Toast notifications provide lightweight, non-intrusive feedback about important changes. Textual's `self.notify()` method provides built-in toast support with severity levels.

## Requirements

1. **State tracking**: Store `_previous_prs: dict[str, PullRequest]` on App (keyed by PR node ID) after each successful fetch
2. **New PR detection**: Compare current PR IDs against previous set; notify for PRs not seen before
3. **CI transition detection**: Compare `ci_status` field between previous and current for each PR; notify on transitions (e.g., pending -> success, pending -> failure)
4. **Review change detection**: Compare `review_decision` field between previous and current; notify on transitions (e.g., None -> APPROVED, None -> CHANGES_REQUESTED)
5. **Notification format**: Use `self.notify(message, severity=...)` with appropriate severity: "information" for new PRs, "warning" for CI failures/changes requested, "information" for CI success/approved
6. **Rate limiting**: Cap at 5 notifications per refresh cycle to avoid overwhelming the user
7. **Skip first load**: Do not generate notifications on the initial data fetch (no previous state to compare)
8. **Tests**: Test change detection logic, notification capping, skip-first-load behavior
