# T43 - PR Sort Options (sort within groups by age, CI, review, size)

## Objective

Sort PRs within each group by configurable criteria: age newest/oldest, CI failing first, review changes-requested first, or size smallest first. Activation via command palette (no dedicated keybinding). Current sort mode shown in Footer.

## Background

Different workflows benefit from different sort orders. A developer triaging CI failures wants failing PRs at the top; a reviewer wanting quick wins wants smallest PRs first. Sort should apply within each group independently, preserving the existing approved-by-me sort-to-bottom behavior as a secondary sort.

## Requirements

1. **Sort modes**: Define sort options: `age_newest` (default), `age_oldest`, `ci_failing_first`, `review_changes_first`, `size_smallest`
2. **Reactive attribute**: Add `_sort_mode` reactive to `PRListWidget`
3. **Sort method**: Add `_sort_prs(prs: list[PullRequest]) -> list[PullRequest]` method implementing each sort mode
4. **Sort order**: Apply primary sort by selected mode, then secondary sort pushing approved-by-me PRs to bottom (existing behavior)
5. **Command palette**: Add "Sort PRs by..." commands to the command palette provider (depends on T40)
6. **Footer display**: Show current sort mode in the Footer widget
7. **Rebuild on change**: Watch `_sort_mode` reactive and call `_rebuild_list` when it changes
8. **Tests**: Test each sort mode produces correct ordering, test approved-by-me secondary sort preserved
