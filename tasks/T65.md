## Objective

Track which PRs have new comments since the user last viewed them and show a visual "unread" indicator on PR rows.

## Background

When monitoring many PRs, it is hard to know which ones have new activity. By persisting the last-viewed timestamp per PR in local state and comparing it to the latest comment timestamp, the dashboard can highlight PRs with new comments.

## Requirements

1. **Local state tracking**: Store `last_viewed_at` timestamp per PR (keyed by PR URL or ID) in local state file
2. **Update on view**: Update `last_viewed_at` when a PR is selected in the detail pane
3. **Unread detection**: Compare `last_viewed_at` against `updatedAt` or latest comment timestamp to determine unread status
4. **Visual indicator**: Show a dot or badge (e.g., "NEW") on PR rows with unread comments
5. **Tests**: Test unread detection logic, state persistence, and indicator rendering
