## Objective

Visually distinguish PRs the current user has already approved, with green background highlighting and sort-to-bottom ordering within each group.

## Background

When monitoring many PRs, it's hard to tell at a glance which ones still need the user's attention vs. which they've already approved. This adds two cues: a subtle green background on approved rows and sorting so approved PRs sink to the bottom of each group, keeping unapproved PRs (needing action) at the top.

Only applies to reviewer groups (direct_reviewer, team_reviewer, mentioned, label). The "authored" / "My PRs" group is excluded since the user wouldn't review their own PR. The green highlight applies whenever the user has personally approved, regardless of the PR's overall review status (e.g., even if another reviewer requested changes).

## Requirements

1. Add `is_approved_by(username: str) -> bool` method to `PullRequest` model ŌĆö returns `True` if any reviewer with matching login has state `"APPROVED"`
2. Pass the current user's username (`AppConfig.username`) into `PRListWidget.update_data()`
3. In `_rebuild_list`, sort PRs within each non-authored group so approved-by-me PRs appear at the bottom (`False < True` stable sort)
4. Add `approved_by_me: bool` flag to `PRRow`, apply `pr-row-approved` CSS class when True
5. Add `.pr-row-approved` CSS rule with subtle green background in `app.tcss`
6. Skip all approved-by-me logic for groups where `group_type == "authored"`
