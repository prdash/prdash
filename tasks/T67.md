## Objective

Detect PR dependency chains (stacked PRs) where PRs target other PR branches and visualize the stack.

## Background

Stacked PRs (where PR B targets PR A's branch instead of main) are a common workflow for large features. Detecting and visualizing these chains helps users understand merge order and identify which PRs in a stack are ready to merge.

## Requirements

1. **Detection**: Identify stacked PRs by matching `baseRefName` of one PR to `headRefName` of another within the same repo
2. **Data model**: Add stack relationship fields (parent PR, child PRs) to `PullRequest` model
3. **Visualization**: Show stack indicator in PR list (e.g., "2/3 in stack") and tree view in detail pane
4. **Ordering**: Within a stack, show PRs in dependency order (base first)
5. **Tests**: Test stack detection logic with various chain configurations
