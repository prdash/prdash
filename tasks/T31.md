## Objective

Add `--version` and `--update` CLI flags to the `ghrd` command so end users can check their installed version and self-update to the latest release.

## Background

The `ghrd` command exists (T24) but has no CLI flags. Users installing via `uv tool install git+https://...` or `pipx install git+https://...` need a way to confirm their version and update without remembering the install-tool-specific upgrade command. Currently there is no argparse or any CLI argument handling — `main()` is purely procedural.

## Requirements

1. Add `argparse` to `__main__.py` with `--version`/`-V` and `--update` flags
2. `ghrd --version` prints version from `importlib.metadata.version("gh-review-dashboard")` and exits (fallback to `"dev"` if metadata unavailable)
3. `ghrd --update` detects install method and runs the appropriate upgrade command:
   - **uv tool**: detected via `uv tool list` output containing `gh-review-dashboard` → runs `uv tool upgrade gh-review-dashboard`
   - **pipx**: detected via `pipx list --short` output containing `gh-review-dashboard` → runs `pipx upgrade gh-review-dashboard`
   - **pip fallback**: runs `sys.executable -m pip install --upgrade gh-review-dashboard`
4. `--version` and `--update` are handled before auth/config/wizard (no GitHub token needed)
5. Running `ghrd` with no flags behaves exactly as today (no regression)
6. Install-method detection and upgrade logic live in a separate `updater.py` module for testability
7. Handle `FileNotFoundError` gracefully when `uv`/`pipx` commands are not installed
8. Print what upgrade command is being run before executing it
9. Update README with end-user install instructions (`uv tool install`, `pipx install`) and document the new flags
