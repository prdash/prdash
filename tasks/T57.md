## Objective

Add a keybinding to open the selected PR's changed files in the user's preferred editor.

## Background

After reviewing a PR in the dashboard, users often want to jump directly into the code. This action would use `gh pr diff --name-only` to get the list of changed files and open them in `$EDITOR`, `$VISUAL`, or a configured editor command. Requires the repo to be checked out locally.

## Requirements

1. **Keybinding**: Add a keybinding (e.g., `e`) to open changed files in the user's editor
2. **File list**: Use `gh pr diff <url> --name-only` to get the list of changed files
3. **Editor detection**: Check `$VISUAL`, `$EDITOR`, then fall back to `code` (VS Code); make configurable in TOML
4. **Error handling**: Handle cases where repo is not checked out locally or editor command is not found
5. **Tests**: Test editor detection priority, command construction, and error paths
