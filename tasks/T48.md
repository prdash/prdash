## Objective

Checkout a PR branch directly from the TUI with the `c` keybinding, using the `gh pr checkout` CLI command.

## Background

A common workflow after reviewing a PR in the dashboard is to check out the branch locally for testing or further review. Currently users must switch to a terminal, find the PR number, and run `gh pr checkout` manually. A single keypress to do this from the dashboard saves time and context-switching.

## Requirements

1. **Keybinding**: Add `c` binding to `NavigableListView` (or App) with `show=False` (not shown in footer)
2. **Selected PR**: Get the currently selected PR from the PR list widget
3. **Checkout command**: Run `gh pr checkout {number} --repo {owner}/{name}` via `asyncio.create_subprocess_exec` in a Textual worker
4. **Success feedback**: Show a toast notification on success: "Checked out PR #{number} ({branch_name})"
5. **Error feedback**: Show a warning toast on failure with the error message from stderr
6. **Guard**: If no PR is selected, show an informational toast and do nothing
7. **Non-blocking**: Run the checkout in a Textual worker so the UI remains responsive
8. **Tests**: Test command construction, success/error toast behavior, no-selection guard

## Acceptance Criteria

- [ ] Pressing `c` with a PR selected runs `gh pr checkout` for that PR
- [ ] The checkout runs in a background worker (non-blocking)
- [ ] A success toast is shown after successful checkout
- [ ] A warning toast is shown if the checkout fails
- [ ] An informational toast is shown if no PR is selected
- [ ] The command uses `--repo` flag with the correct owner/name
- [ ] All new/modified code has corresponding tests
- [ ] All existing tests still pass
