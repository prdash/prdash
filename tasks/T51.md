## Objective

Show discussion activity count on each PR row and in the detail pane to help users gauge how active or contentious a PR is.

## Background

Comment count is a useful signal for PR prioritization. A PR with many comments may indicate active discussion, unresolved disagreements, or a complex review. Surfacing this count helps users decide which PRs need attention. GitHub's GraphQL API provides `comments { totalCount }` on PullRequest nodes.

## Requirements

1. **GraphQL query**: Add `comments { totalCount }` to `PR_SEARCH_QUERY` in `queries.py`
2. **Data model**: Add `comment_count: int = 0` field to `PullRequest` model
3. **Response parsing**: Parse `comments.totalCount` from the GraphQL response in `parse_pr_node()`
4. **PR row display**: Show comment count on the metadata line of each PR row (e.g., a speech bubble icon or text like "3 comments"), only when count > 0
5. **Detail pane display**: Show comment count in the detail pane metadata section
6. **Tests**: Test model field, parsing, PR row display (zero and non-zero), detail pane display
