## Objective

Live-poll CI check status for the currently selected PR at a higher frequency than the global refresh, showing real-time progress indicators.

## Background

When waiting on CI, users want to see check progress update in near-real-time without manually refreshing. A focused polling mode for the selected PR's status checks can update every few seconds, showing individual check names, statuses, and elapsed time.

## Requirements

1. **Focused polling**: When viewing a PR's detail, poll its CI status checks at a configurable higher frequency (e.g., every 10s)
2. **Visual indicators**: Show each check with name, status icon (pending/running/pass/fail), and elapsed time
3. **Auto-stop**: Stop focused polling when the user navigates away from the PR or all checks complete
4. **GraphQL query**: Use `statusCheckRollup` or `commits.nodes.commit.statusCheckRollup` for check details
5. **Tests**: Test polling lifecycle (start, update, stop) and status rendering
