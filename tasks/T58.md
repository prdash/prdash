## Objective

Add a notifications query group showing unread GitHub notifications with the ability to mark them as read.

## Background

GitHub notifications complement PR review requests by surfacing mentions, CI failures, and other events. Integrating the notifications REST API (`/notifications`) alongside the existing GraphQL PR queries provides a unified view of items requiring attention.

## Requirements

1. **Notifications query**: Fetch unread notifications via GitHub REST API (`GET /notifications`) using the existing auth token
2. **Query group**: Add a "Notifications" collapsible group in the PR list displaying notification subjects
3. **Mark as read**: Keybinding to mark a notification as read (`PATCH /notifications/threads/{id}`)
4. **Filtering**: Filter notifications to show only PR-related or all types based on config
5. **Tests**: Test notification fetching, rendering, and mark-as-read API calls
