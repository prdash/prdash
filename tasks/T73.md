## Objective

Send native OS-level notifications when new PRs appear, in addition to in-app toast notifications.

## Background

When the dashboard is running in the background or on another workspace, in-app toasts are not visible. Native OS notifications (via `osascript` on macOS, `notify-send` on Linux) ensure users are alerted to new PRs requiring their attention even when not actively looking at the dashboard.

## Requirements

1. **macOS notifications**: Use `osascript -e 'display notification ...'` to send native notifications
2. **Linux notifications**: Use `notify-send` to send desktop notifications
3. **Trigger**: Send a notification when a refresh detects new PRs that were not in the previous data set
4. **Config toggle**: Add `desktop_notifications = true/false` to TOML config (default false)
5. **Tests**: Test notification command construction per platform and new-PR detection logic
