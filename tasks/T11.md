## Objective

Add robust error handling across the application so that failures are surfaced gracefully in the UI rather than crashing the app.

## Requirements

1. API errors: show error message in the UI, don't crash
2. Auth failures: clear message directing user to `gh auth login`
3. Network timeouts: configurable timeout, retry with backoff or show error
4. No PRs: display "No pull requests found" in empty groups
5. Config errors: clear message at startup if config is invalid or missing

### Network Timeout & Retry

In `client.py`:
```python
class GitHubClient:
    def __init__(self, token: str, timeout: float = 30.0):
        self._client = httpx.AsyncClient(
            ...
            timeout=httpx.Timeout(timeout),
        )

    async def execute_query(self, query: str, variables: dict) -> dict:
        try:
            response = await self._client.post("/graphql", json={"query": query, "variables": variables})
            response.raise_for_status()
            data = response.json()
            if "errors" in data:
                raise GitHubAPIError(data["errors"])
            return data["data"]
        except httpx.TimeoutException:
            raise NetworkError("Request timed out. Check your network connection.")
        except httpx.HTTPStatusError as e:
            if e.response.status_code == 401:
                raise AuthError("GitHub token expired. Run 'gh auth login' to re-authenticate.")
            raise GitHubAPIError(f"GitHub API returned {e.response.status_code}")
```

### Custom Exceptions

```python
class GitHubAPIError(Exception):
    """Raised when the GitHub API returns an error."""
    pass

class NetworkError(Exception):
    """Raised for network-level failures."""
    pass
```

### UI Error Display

In `app.py`, wrap refresh in error handling:
```python
async def refresh_data(self) -> None:
    try:
        groups = await self.github_client.fetch_all_groups(self.config)
        self.query_one(PRListWidget).update_data(groups, seen_ids=self._seen_pr_ids)
        self._seen_pr_ids = ...
    except AuthError as e:
        self.notify(str(e), severity="error")
    except (GitHubAPIError, NetworkError) as e:
        self.notify(str(e), severity="warning")
```

Use Textual's `self.notify()` for transient error messages, or display a persistent error banner for critical failures (auth).

## Acceptance Criteria

- [ ] API errors are shown in the UI (via `notify` or error banner), app does not crash
- [ ] Auth failures show a message directing user to run `gh auth login`
- [ ] Network timeouts are caught and shown as a user-friendly message
- [ ] httpx client has a configurable timeout (default 30s)
- [ ] Empty groups display "No pull requests found" instead of blank space
- [ ] Config errors at startup produce a clear, actionable error message
- [ ] GraphQL-level errors (in response body) are detected and reported
- [ ] The app remains functional after transient errors (can retry on next refresh)
