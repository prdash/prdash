## Objective

Show `+N/-M` line change counts on each PR row to help users gauge PR size at a glance.

## Background

PR size is an important signal for reviewers. Small PRs are quicker to review and easier to reason about, while large PRs may need more time. GitHub's API provides `additions` and `deletions` fields on PullRequest nodes. Displaying these as colored `+N/-M` indicators helps users prioritize their review queue.

## Requirements

1. **GraphQL query**: Add `additions` and `deletions` fields to `PR_SEARCH_QUERY` in `queries.py`
2. **Data model**: Add `additions: int = 0` and `deletions: int = 0` fields to `PullRequest` model
3. **Computed property**: Add `total_changes: int` property returning `additions + deletions`
4. **Response parsing**: Parse `additions` and `deletions` from the GraphQL response in `parse_pr_node()`
5. **PR row display**: Show `+{additions}/-{deletions}` on the metadata line of each PR row, with green for additions and red for deletions
6. **Compact format**: For large numbers, use compact format (e.g., `+1.2k/-350`)
7. **Tests**: Test model fields, computed property, parsing, compact formatting, widget display
