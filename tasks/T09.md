## Objective

Allow the user to open the currently selected PR in their default web browser by pressing `Enter`.

## Requirements

1. `Enter` on a selected PR row opens the PR URL in the default browser
2. `Enter` on a group header still toggles collapse/expand (from T08)
3. Use Python's `webbrowser.open()`

## Implementation Details

### Files to Modify

- **`src/gh_review_dashboard/app.py`** or **`src/gh_review_dashboard/widgets/pr_list.py`** - Handle Enter action based on context

### Logic

The `Enter` key behavior depends on what is currently selected:

```python
import webbrowser

def action_select(self) -> None:
    """Handle Enter key press."""
    current = self.get_current_item()
    if current is None:
        return
    if isinstance(current, GroupHeader):
        current.toggle_collapse()
    elif isinstance(current, PRRow):
        webbrowser.open(current.pull_request.url)
```

### Keybinding

Add or update in the app or PR list widget:
```python
Binding("enter", "select", "Open PR / Toggle Group"),
```

## Acceptance Criteria

- [ ] Pressing `Enter` on a PR row opens the PR's GitHub URL in the default browser
- [ ] Pressing `Enter` on a group header toggles collapse/expand
- [ ] The correct URL is opened (matches the selected PR)
- [ ] The app remains responsive after opening the browser (non-blocking)
- [ ] Footer hint shows the Enter action
