## Objective

Show how long a PR has been waiting for review and display a "stale" badge for PRs with no activity beyond a configurable threshold.

## Background

Teams need visibility into PRs that are languishing without review. Displaying the time since a review was requested (or since the last activity) helps prioritize which PRs to review first. A "stale" badge draws attention to PRs that have been idle too long.

## Requirements

1. **Wait time display**: Show human-readable time since review was requested (e.g., "2d 4h") in PR row and detail pane
2. **Stale badge**: Display a "STALE" badge on PRs with no activity (comments, commits, reviews) in the last N days (configurable, default 3)
3. **GraphQL data**: Use `reviewRequests.nodes.createdAt` and `updatedAt` fields to compute wait times
4. **Config**: Add `stale_threshold_days` to TOML config
5. **Tests**: Test wait time calculation, stale detection, and badge rendering
