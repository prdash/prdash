## Objective

Add toggleable layout modes: split-pane (current default), full-width list, and full-width detail view.

## Background

Depending on terminal width and user preference, different layouts may be more effective. A narrow terminal benefits from full-width modes, while wide terminals work well with the split pane. Users should be able to cycle between modes.

## Requirements

1. **Layout modes**: Support three modes: split-pane (default), list-only (full width), detail-only (full width)
2. **Keybinding**: Add a keybinding to cycle through layout modes
3. **CSS classes**: Toggle CSS classes on the main container to show/hide panes
4. **Persistence**: Remember last-used layout mode in config or local state
5. **Tests**: Test layout mode cycling and CSS class application
