## Objective

Add an `Input` widget that filters visible PRs by substring match across title, author, repo slug, and PR number.

## Background

When many PRs are loaded across multiple groups, users need a quick way to narrow down the list to find a specific PR. A `/` key to activate search is a common convention in terminal applications (vim, less, etc.). The filter should work across all visible groups, hiding PRs that don't match and collapsing groups that become empty.

## Requirements

1. **Input widget**: Add a Textual `Input` to `PRListWidget.compose()`, initially hidden (`display: none`)
2. **Activation**: `/` keybinding on App or PRListWidget shows the Input and focuses it
3. **Filtering**: On `Input.Changed`, apply case-insensitive substring matching against PR title, author login, repo slug (`owner/name`), and PR number (as string)
4. **Group handling**: Hide groups that have zero matching PRs; show all groups when filter is cleared
5. **Dismissal**: Escape hides the Input, clears the filter, and restores full list; Enter hides the Input but keeps the filter active
6. **Visual indicator**: When a filter is active (Input hidden but filter text non-empty), show a subtle indicator (e.g., filter text in Footer or a label)
7. **Tests**: Test filtering by title, author, repo, number; test empty groups hidden; test Escape clears; test Enter preserves
