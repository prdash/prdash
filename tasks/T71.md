## Objective

Add a Textual Sparkline widget in the detail pane showing commit and comment activity over time for the selected PR.

## Background

A visual activity graph helps reviewers quickly gauge whether a PR is actively being worked on or has gone stale. The sparkline shows data points over the PR's lifetime, with each point representing commits or comments in a time bucket.

## Requirements

1. **Data collection**: Aggregate commit and comment timestamps from PR data into time-bucketed counts (e.g., per day)
2. **Sparkline widget**: Use Textual's `Sparkline` widget to render the activity graph in the detail pane
3. **Time range**: Show activity from PR creation to now, with appropriate bucket sizing based on PR age
4. **Tooltip/label**: Show a label like "Activity (last 14d)" above the sparkline
5. **Tests**: Test time-bucket aggregation logic and sparkline data generation
