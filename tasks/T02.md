## Objective

Build a TOML-based configuration system with Pydantic models that provides sensible defaults and validates user-provided configuration.

## Requirements

1. Define Pydantic models for all configuration sections
2. Support loading from `~/.config/gh-review-dashboard/config.toml`
3. Provide sensible defaults so the app works with minimal or no config file
4. Validate configuration on load and raise clear errors for invalid config

## Implementation Details

### Files to Create

- **`src/gh_review_dashboard/config.py`**

### Pydantic Models

```python
from enum import Enum

class QueryGroupType(str, Enum):
    DIRECT_REVIEWER = "direct_reviewer"
    TEAM_REVIEWER = "team_reviewer"
    MENTIONED = "mentioned"
    AUTHORED = "authored"
    LABEL = "label"

class QueryGroupConfig(BaseModel):
    type: QueryGroupType
    name: str                      # Display name for the group
    labels: list[str] = []         # Only used when type == "label"
    enabled: bool = True

class RepoConfig(BaseModel):
    org: str                       # GitHub org or owner
    name: str                      # Repository name

class AppConfig(BaseModel):
    repo: RepoConfig
    username: str                  # GitHub username
    team_slugs: list[str] = []    # Team slugs for team reviewer queries
    poll_interval: int = 300       # Seconds between auto-refreshes (default 5 min)
    query_groups: list[QueryGroupConfig] = [
        # Default groups - all 5 types enabled
        QueryGroupConfig(type="direct_reviewer", name="Requested Reviewer"),
        QueryGroupConfig(type="team_reviewer", name="Team Reviewer"),
        QueryGroupConfig(type="mentioned", name="Mentioned/Involved"),
        QueryGroupConfig(type="authored", name="My PRs"),
        QueryGroupConfig(type="label", name="Labeled", labels=[], enabled=False),
    ]
```

### Config File Path

```python
CONFIG_DIR = Path.home() / ".config" / "gh-review-dashboard"
CONFIG_FILE = CONFIG_DIR / "config.toml"
```

### Load Function

```python
def load_config(path: Path | None = None) -> AppConfig:
    """Load config from TOML file. Uses defaults if file doesn't exist."""
```

- If no path provided, use `CONFIG_FILE`
- If file doesn't exist, return `AppConfig` with defaults (will require `repo` and `username` to be set, so raise a clear error directing user to create config)
- Parse TOML using `tomllib` (stdlib in 3.11+)
- Validate with Pydantic and return `AppConfig`

## Acceptance Criteria

- [ ] `config.py` defines `AppConfig`, `RepoConfig`, `QueryGroupConfig`, and `QueryGroupType`
- [ ] `load_config()` returns a valid `AppConfig` from a sample TOML file
- [ ] When no config file exists, `load_config()` raises a clear error with instructions to create a config file
- [ ] Invalid config values (e.g., unknown group type, negative poll_interval) produce clear Pydantic validation errors
- [ ] Default query groups include all 5 types with sensible names
