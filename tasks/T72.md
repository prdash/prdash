## Objective

Serve the dashboard in a web browser via `textual-serve`, enabled with a `--web` CLI flag.

## Background

`textual-serve` allows Textual apps to be served over HTTP and rendered in a web browser. This is useful for shared team dashboards, remote access, or environments without a terminal emulator. The `--web` flag starts the app in serve mode on a configurable port.

## Requirements

1. **CLI flag**: Add `--web` flag and optional `--port` argument to the CLI
2. **textual-serve integration**: When `--web` is passed, use `textual-serve` to serve the app instead of running in the terminal
3. **Dependency**: Add `textual-serve` as an optional dependency (`prdash[web]`)
4. **Graceful fallback**: If `textual-serve` is not installed, show a clear error message with install instructions
5. **Tests**: Test CLI flag parsing and serve mode initialization (mock the serve call)
