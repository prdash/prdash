## Objective

Add issue query groups (assigned issues, mentioned issues) alongside PR groups, using the same list/detail UI pattern.

## Background

Users often track both PRs and issues. Adding issue groups to the dashboard provides a unified view of all items requiring attention. GitHub's search API supports issue queries with the same `is:issue` qualifier.

## Requirements

1. **Query groups**: Add "Assigned Issues" and "Mentioned Issues" query groups using `is:issue assignee:@me` and `is:issue mentions:@me`
2. **Data model**: Add `Issue` model (or extend existing models) with issue-specific fields (state, labels, milestone)
3. **List rendering**: Render issues in collapsible groups alongside PR groups, with appropriate icons
4. **Detail pane**: Show issue details (body, labels, assignees, milestone) in the detail pane when an issue is selected
5. **Tests**: Test issue query construction, model parsing, and rendering
