## Objective

Show Textual's built-in `LoadingIndicator` overlay on the PR list during data refresh to provide visual feedback that a fetch is in progress.

## Background

Currently, when the dashboard refreshes data (either auto-refresh or manual `r`), there is no visual indication that a fetch is happening. Users may think the app is frozen or unresponsive. Textual widgets have a built-in `loading` property that overlays a `LoadingIndicator` animation when set to `True`.

## Requirements

1. **Loading on refresh**: Set `pr_list_widget.loading = True` at the start of `refresh_data`
2. **Loading cleared on success**: Set `pr_list_widget.loading = False` after PR list is updated with new data
3. **Loading cleared on error**: Set `pr_list_widget.loading = False` in error handling paths (both API errors and unexpected exceptions)
4. **Initial load**: Show loading state on first mount before any data has arrived
5. **Detail pane**: Optionally show loading on the detail pane as well during refresh
6. **Tests**: Verify loading is True during fetch, False after success, False after error
