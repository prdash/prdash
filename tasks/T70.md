## Objective

Support multiple config files (e.g., work.toml, personal.toml) with a CLI `--config` flag and an in-app profile switcher.

## Background

Users who contribute to multiple organizations often need different query groups, tokens, and settings for each context. Named config profiles let users switch between configurations without manual file editing.

## Requirements

1. **CLI flag**: Add `--config <path>` CLI argument to specify an alternate config file
2. **Profile discovery**: Auto-discover config profiles in `~/.config/prdash/` directory (any `.toml` file)
3. **Profile switcher**: Add a command palette action or keybinding to switch between discovered profiles at runtime
4. **Reload on switch**: Reload queries and refresh data when switching profiles
5. **Tests**: Test profile discovery, CLI flag parsing, and config reload behavior
