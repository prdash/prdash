## Objective

Leverage Textual's built-in theme system with its bundled themes, persist the user's theme choice in the TOML config file, and expose theme switching via the command palette.

## Background

Textual ships with approximately 10 built-in themes (e.g., `textual-dark`, `textual-light`, `monokai`, `dracula`, `nord`, `gruvbox`, `catppuccin-mocha`, etc.). Users should be able to pick a theme that suits their terminal and preferences, with the choice persisted across sessions.

## Requirements

1. **Config field**: Add `theme: str = "textual-dark"` to `AppConfig` in `config.py`
2. **Apply on startup**: Set `self.theme` in App's `__init__` or `on_mount` from the loaded config
3. **Command palette**: Add "Change Theme: {name}" commands for each available Textual theme to the command palette provider
4. **Persist on change**: When a theme is selected via the palette, update the config and save to disk
5. **Validation**: If the configured theme name is invalid (e.g., user edited TOML manually), fall back to the default theme
6. **Tests**: Test config field default, theme application, invalid theme fallback, persistence

## Acceptance Criteria

- [ ] `AppConfig` has a `theme` field defaulting to `"textual-dark"`
- [ ] The configured theme is applied when the app starts
- [ ] Theme can be changed via the command palette
- [ ] Theme selection is persisted to the TOML config file
- [ ] Invalid theme names fall back to the default theme
- [ ] All new/modified code has corresponding tests
- [ ] All existing tests still pass
