## Objective

Add a subtle background tint to group header rows when focused/highlighted in the PR list, so users can see which row the cursor is on.

## Background

When navigating the PR list with j/k or arrow keys, moving focus to a collapsible group header currently shows no visual indication of the focused row. PR rows get Textual's default ListView highlight, but group headers blend in with their static `$surface` background. Users need a visible cue to know when a group header is focused.

## Requirements

1. Add CSS styling so that a `GroupHeaderItem` shows a subtle background tint when it is the highlighted item in the ListView
2. Use Textual's `ListView > ListItem.--highlight` selector (or equivalent) targeting `GroupHeaderItem`
3. The highlight should be a slightly lighter/darker shade than the default `$surface` background — visible but not distracting
4. The highlight must clear when focus moves to another item
5. PR row highlighting must continue to work as before
