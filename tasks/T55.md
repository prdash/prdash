## Objective

Allow users to define custom shell commands in TOML config that can be executed against the selected PR with data interpolation.

## Background

Different teams have unique workflows (e.g., deploying a PR branch, running local tests, posting to Slack). A templated action system lets users define commands like `gh pr checkout {number}` or `open https://deploy-preview-{number}.example.com` in their config file, bound to custom keybindings.

## Requirements

1. **Config schema**: Add `[[actions]]` table array in TOML config with fields: `name`, `key`, `command` (template string)
2. **Template interpolation**: Support `{number}`, `{url}`, `{repo_slug}`, `{branch}`, `{title}` placeholders in command strings
3. **Execution**: Run interpolated command via subprocess, show output/errors as toast notifications
4. **Validation**: Validate action definitions on config load; warn on duplicate key bindings
5. **Tests**: Test template interpolation, config parsing, and conflict detection
