## Objective

Distinguish required vs optional reviewers and surface the `reviewDecision` field to help answer "what's blocking this PR?"

## Background

Branch protection rules and CODEOWNERS files create required review requirements. Showing which reviewers are required (vs. optional) and the overall review decision (approved, changes requested, review required) helps PR authors and reviewers understand what's needed to unblock a merge.

## Requirements

1. **GraphQL query**: Fetch `reviewDecision` on PullRequest and `isRequired` on review request nodes (if available)
2. **Data model**: Add `review_decision` field to `PullRequest` and required/optional distinction to `Reviewer` model
3. **Detail pane**: Show review decision status and visually distinguish required vs optional reviewers
4. **Blocking summary**: Display a concise "blocking" summary (e.g., "Awaiting 1 required review") in the detail pane
5. **Tests**: Test review decision parsing and blocking summary logic
