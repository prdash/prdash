## Objective

Allow left/right arrow keys to collapse and expand group headers in the PR list.

## Background

Currently, collapsing/expanding a group header requires pressing Enter. Users expect left arrow to collapse and right arrow to expand, which is a common tree-navigation pattern. The left/right keys should only toggle state — no cursor movement beyond the toggle action. If the group is already in the target state (e.g., right arrow on an already-expanded group), do nothing.

## Requirements

1. Add `right` key handling on `NavigableListView` (or `PRList`): if the focused item is a collapsed `GroupHeaderItem`, expand it
2. Add `left` key handling: if the focused item is an expanded `GroupHeaderItem`, collapse it
3. If the group is already in the target state, do nothing (no cursor movement)
4. Reuse existing collapse/expand logic from `on_list_view_selected()` — extract shared toggle method if needed
5. Enter key must continue to work as a toggle (existing behavior preserved)
6. j/k and up/down navigation must remain unaffected
