## Objective

Fix the indentation of wrapped PR title lines in the PR list pane so they align consistently with the first line.

## Background

When a PR title is long enough to wrap to a second line in the PR list widget, the wrapped text loses its indentation. The first line is properly indented after the new-item marker (`●` or 2 spaces), but wrapped text returns to column 0 (plus CSS padding). This happens because the indent is done with inline spaces in a single `Static` widget, and Textual doesn't support a `text-indent` CSS property.

## Requirements

1. Restructure `PRRow.compose()` to use a `Horizontal` container that separates the fixed-width marker column (2 chars) from the content column
2. The content Static gets `width: 1fr` so Textual's layout system handles wrapping within the content area
3. All wrapped title lines must align with the start of the title text on the first line
4. Preserve existing CSS classes and behavior: `pr-row-label`, `pr-row-new`, `pr-row-approved`
5. Preserve the new-item marker (`●`) for unseen PRs and blank space for seen PRs

## Files

| File | Changes |
|------|---------|
| `src/gh_review_dashboard/widgets/pr_list.py` | Add `Horizontal` import; rewrite `PRRow.compose()` to use Horizontal layout with separate marker Static |
| `src/gh_review_dashboard/app.tcss` | Add `.pr-row-container` and `.pr-row-marker` CSS rules; update `.pr-row-label` to use `width: 1fr` and adjust padding |
| `tests/test_pr_list.py` | Update ~10 `query_one(Static)` calls to `query_one(".pr-row-label", Static)`; update `●` marker assertions to query `.pr-row-marker` |
