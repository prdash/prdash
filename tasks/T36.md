## Objective

Add a visible "DRAFT" badge to draft PRs — an inline cyan badge on the PR list metadata line plus a Draft indicator in the detail pane metadata header.

## Background

Draft PRs are a common GitHub feature, but the dashboard currently has no visual indicator for draft state. Users scanning the PR list cannot distinguish draft PRs from ready-for-review PRs at a glance. Adding a badge consistent with the existing CI/Review status badge pattern provides immediate visibility.

## Requirements

1. **GraphQL query**: Add `isDraft` field to the PR search query in `queries.py`
2. **Data model**: Add `is_draft: bool = False` field to the `PullRequest` model in `models.py`
3. **Response parsing**: Parse `isDraft` from GraphQL response nodes in `client.py` (in `parse_pr_node` or equivalent)
4. **PR list badge**: Show `[cyan]DRAFT[/cyan]` badge on the metadata line (between age and CI status) when `is_draft` is True — matching existing badge styling pattern
5. **Detail pane badge**: Show a "Draft" indicator in the detail pane metadata header when the selected PR is a draft
6. **Tests**: Unit tests for model field, parser extraction, and badge rendering
