## Objective

Implement auto-refresh on a configurable timer, manual refresh via keybinding, and visual indicators for newly appeared PRs.

## Requirements

1. Auto-refresh using a Textual timer based on `poll_interval` from config
2. Manual refresh via `r`/`R` keybinding
3. Track seen PR IDs and visually mark new ones
4. Refresh should not disrupt the user's current selection

## Implementation Details

### Files to Modify

- **`src/gh_review_dashboard/app.py`** - Timer setup, refresh orchestration, keybinding
- **`src/gh_review_dashboard/widgets/pr_list.py`** - New-item visual indicator, preserve selection on data update

### Auto-Refresh Timer

In `app.py`:
```python
def on_mount(self) -> None:
    self.set_interval(self.config.poll_interval, self.refresh_data)

async def refresh_data(self) -> None:
    """Fetch fresh data from GitHub and update the UI."""
    groups = await self.github_client.fetch_all_groups(self.config)
    pr_list = self.query_one(PRListWidget)
    pr_list.update_data(groups, seen_ids=self._seen_pr_ids)
    # Update seen IDs after displaying
    self._seen_pr_ids = {pr.id for group in groups for pr in group.pull_requests}
```

### Manual Refresh

```python
BINDINGS = [
    ...
    Binding("r", "refresh", "Refresh"),
]

async def action_refresh(self) -> None:
    await self.refresh_data()
```

### New-Item Indicator

Track PR IDs that were present in the previous refresh:

```python
class ReviewDashboardApp(App):
    _seen_pr_ids: set[str] = set()
```

In `pr_list.py`, when rendering PR rows:
- If a PR's ID is not in `seen_ids`, mark it as new
- Visual indicator: bold text, or a `*` prefix, or a distinct color
- On first load, all PRs are "seen" (no indicators)

### Preserve Selection

When `update_data()` is called:
1. Remember the currently selected PR's ID
2. Rebuild the list with new data
3. If the previously selected PR still exists, re-select it
4. If it no longer exists, select the first PR in the list

## Acceptance Criteria

- [ ] PRs auto-refresh on the interval specified in config (`poll_interval`)
- [ ] Pressing `r` or `R` triggers an immediate refresh
- [ ] PRs that are new since the last refresh are visually distinct (bold, badge, or color)
- [ ] On first load, no PRs are marked as new
- [ ] The currently selected PR is preserved across refreshes (if it still exists)
- [ ] The UI remains responsive during refresh (async, non-blocking)
- [ ] Footer shows the refresh keybinding hint
