## Objective

Change the default behavior from single-repo scoping to showing all PRs across all repos (like `github.com/pulls/review-requested`), with optional repo filters to constrain scope. Display repo information in both the PR list and detail pane.

## Background

The dashboard currently hardcodes a single `repo:org/name` in every GitHub search query. This limits users to monitoring one repo at a time. GitHub's own review-requested page shows PRs across all repos a user has access to. We want to match that behavior by default, while allowing users to optionally filter by one or more repos.

## Requirements

### 1. Config model changes (`config.py`)

- **Replace** `repo: RepoConfig` (single, required) with `repos: list[str]` (optional list of `"org/name"` strings, default `[]`)
- Empty list means "all repos" (no `repo:` prefix in queries)
- Non-empty list scopes queries to those repos
- Add a **field validator** ensuring each entry matches `org/name` format
- Add a **`model_validator(mode="before")`** for backward compatibility: if the old `{"repo": {"org": "x", "name": "y"}}` format is present, migrate to `{"repos": ["x/y"]}`
- `RepoConfig` class can be removed (or kept internally if useful)
- Add helper: `get_org_from_repos(repos: list[str]) -> str | None` â€” extracts org from first entry

### 2. Config serialization (`config.py`)

- Update `_serialize_config_toml` to write `repos = ["org/repo1", "org/repo2"]` instead of the `[repo]` block
- Update `load_config` error message to remove reference to required `[repo]` section

### 3. GraphQL query changes (`queries.py`)

- Add `repository { nameWithOwner }` to `PR_SEARCH_QUERY` inside `... on PullRequest` block
- Update `build_search_query`:
  - `repos=[]` â†’ prefix is `"is:pr is:open"` (no repo filter)
  - `repos=["a/b", "c/d"]` â†’ one query per repo per group type
  - `team_reviewer` with empty repos â†’ return `[]` (skip, since org is unknown)

### 4. PR model changes (`models.py`)

- Add `repo_slug: str = ""` field to `PullRequest` (frozen model, default empty string)
- Update `parse_pr_node` to extract: `(node.get("repository") or {}).get("nameWithOwner", "")`

### 5. PR list display (`widgets/pr_list.py`)

- Update `PRRow.compose` to include `self.pr.repo_slug` in the label, e.g.:
  ```
  â—Ź [*][+] org/repo  PR title  author  3d
  ```

### 6. Detail pane display (`widgets/detail_pane.py`)

- Update `_format_metadata` to prepend `repo_slug` when present:
  ```
  org/repo PR title (#123)
  by author | opened 3d ago
  https://github.com/org/repo/pull/123
  ```

### 7. App subtitle (`app.py`)

- In `on_mount`, set `self.sub_title` based on `config.repos`:
  - Non-empty â†’ comma-joined repo list
  - Empty â†’ `"All repositories"`

### 8. Setup wizard changes (`screens/setup_wizard.py`)

- `RepoScreen` (step 1): Add a "Skip (show all repos)" button alongside Next
- If skipped, leave org/repo empty in wizard state
- Final save step: build `repos = [f"{org}/{repo_name}"]` if both set, else `[]`
- Only call `detect_team_slugs` if org is available

### 9. Settings screen changes (`screens/settings.py`)

- Replace separate org/name `Input` fields with a single "Repos" `Input` (comma-separated `org/name` entries)
- Pre-fill from `", ".join(self._config.repos)`
- Parse back to `list[str]` on save, validate format

### 10. Client changes (`github/client.py`)

- In `fetch_group`: if `build_search_query` returns `[]`, return empty `QueryGroupResult` (no error, no API call)
