## Objective

Make PR list rows more readable by replacing cryptic status symbols with colored labels, switching to a multi-line row layout, and adding visual separators between groups.

## Background

The PR list currently renders each PR as a dense single line of plain text with cryptic symbols like `[!][?][~]` for CI and review status. Users can't tell what these mean without memorizing the mapping. Rows run together with no spacing or visual delineation between groups, making the list hard to scan.

Current format:
```
  [*][+] Fix auth bug  @alice  2d
● [!][x] Refactor API handler  @bob  5h
```

## Requirements

1. **Replace status icon maps with colored Rich markup labels** — Textual's `Static` widget supports Rich console markup. Replace `_CI_ICONS` and `_REVIEW_ICONS` with:
   - CI: `[green]CI:pass[/green]`, `[red]CI:fail[/red]`, `[yellow]CI:pend[/yellow]`, `[dim]CI:--[/dim]`
   - Review: `[green]Rev:ok[/green]`, `[red]Rev:chg[/red]`, `[yellow]Rev:pend[/yellow]`, `[dim]Rev:--[/dim]`

2. **Multi-line PR rows** — Each PR gets 2 lines in a single `Static` with `\n`:
   - Line 1: `{new_marker}{title}`
   - Line 2: `  [dim]@{author} · {age}[/dim] · {ci_label} · {review_label}`

3. **Escape PR titles and authors** — Use `rich.markup.escape()` on `title` and `author` to prevent `[`/`]` characters from being parsed as Rich markup tags.

4. **Improve group header arrows** — Change `">"` / `"v"` to `"▶"` / `"▼"`.

5. **Add group visual separators** — Use CSS `border-bottom` on `.group-header-label` and `margin-top: 1` on `GroupHeaderItem` (except first group via `.first-group` class) for spacing between groups.

6. **CSS updates** — Add `height: auto` on `.pr-row-label` to ensure 2-line content is fully visible.

7. **Update and add tests** — Existing tests should pass without changes (they check structural properties, not rendered text). Add new tests for colored labels, multi-line rendering, and triangle arrows.

## Acceptance Criteria

- [ ] Status indicators show colored self-documenting labels (e.g., `CI:pass` in green, `CI:fail` in red) instead of `[*]`, `[!]`, etc.
- [ ] Each PR row displays on 2 lines: title on line 1, metadata (author, age, statuses) on line 2
- [ ] Group headers use ▼/▶ arrows and have a visual divider line below them
- [ ] Groups have vertical spacing between them
- [ ] PR titles containing `[` or `]` are rendered correctly (escaped)
- [ ] All existing tests pass without modification
- [ ] New tests cover colored labels, multi-line layout, and triangle arrows
- [ ] Navigation (j/k, Enter collapse/expand, Enter open browser) works correctly with new layout
- [ ] New PR marker (●) still appears correctly on unseen PRs
