## Objective

Add keybindings to approve a PR and merge it directly from the dashboard using the `gh` CLI.

## Background

Power users want to approve and merge PRs without leaving the terminal dashboard. The `gh pr review --approve` and `gh pr merge` commands provide this capability. Merge should require a confirmation modal to prevent accidental merges.

## Requirements

1. **Approve keybinding**: Shell out to `gh pr review --approve <url>` for the selected PR, show toast with result
2. **Merge keybinding**: Show a confirmation modal before executing `gh pr merge <url>`, support merge method selection (merge/squash/rebase)
3. **Error handling**: Surface `gh` CLI errors (not authenticated, merge conflicts, branch protection) as toast notifications
4. **Auto-refresh**: Trigger a data refresh after successful approve or merge to update PR state
5. **Tests**: Test command construction, confirmation modal flow, and error toast display
