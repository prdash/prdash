## Objective

Add a green background highlight to PRs in the "My PRs" (authored) group that are ready to merge, using GitHub's `mergeStateStatus` API field.

## Background

Users want to quickly identify which of their authored PRs can be merged by looking at the dashboard. Currently there's no visual distinction for merge-ready PRs. GitHub's `mergeStateStatus` field on PullRequest returns `CLEAN` when a PR has all required approvals, passing CI, and no merge conflicts.

## Requirements

1. **GraphQL query**: Add `mergeStateStatus` field to `PR_SEARCH_QUERY`
2. **Data model**: Add `merge_state_status: str = "UNKNOWN"` field to `PullRequest`
3. **Computed property**: Add `ready_to_merge` property returning `True` when `merge_state_status == "CLEAN"`
4. **Response parsing**: Parse `mergeStateStatus` from GraphQL response in `parse_pr_node()`
5. **Widget**: Add `ready_to_merge` param to `PRRow`, only set `True` for authored groups
6. **CSS**: Add `.pr-row-ready-to-merge { background: $success 15%; }` rule
7. **Tests**: Model tests for `ready_to_merge` property, parser tests, widget tests for class application
